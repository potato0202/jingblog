Hexo + Github 免費架站筆記 (二) 將網站放到 GitHub

發表於2024-06-16|更新於2024-06-17|Hexo
|閱讀量:

準備部署嘍！ VSCode 與 GitHub 綁定

先註冊 GitHub ，點擊官網連結

讓 VSCode 綁定你的 GitHub 帳號

step01 開啟 VSCode，點擊左下角「大頭圖案」，選擇啟用雲端變更

step02 點選 利用 GitHub 登入

step03 瀏覽器會另開一個網頁，出現下圖，點藍色按鈕授權

step04 瀏覽器頁面出現 對話窗，點擊 「開啟」，之後在 VSCode 「大頭圖案」點擊就可看到有連結帳號了~

在 GitHub 開啟一個網站空間

在 GitHub 開個 repository 儲存庫，上方有個「+」，點擊並選擇「New repository」
命名(例如 000blog)，其他設定不變，點擊「Create repository」
https://github.com/xxxhappy(這裡是你的使用者名稱username)/000blog
設定好的頁面網址先複製，之後會用到~

更改 _config.yml 的 URL

Hexo + GitHub Pages 官方部署說明

要調整 _config.yml 的 URL，進入 _config.yml 頁面
( 不是 themes 底下的 _config.yml 哦~ 是控制全站的 )
# URL
## Set your site url here. For example, if you use GitHub Page, set url as 'https://username.github.io/project'

在控制全站的 _config.yml 檔案，找到 url 改成你的，例如 https://xxxhappy.github.io/000blog

如果你剛剛建立的github儲存庫頁面是 https://github.com/xxxhappy/000blog，
user name 是 xxxhappy ，project 是 000blog，你的 url 就是 https://xxxhappy.github.io/000blog

一鍵部署

官方部署說明也有寫一鍵部屬

  1. 安裝 hexo-deployer-git
    複製 : npm install hexo-deployer-git --save ，並在 VSCode 輸入指令，這是一鍵安裝的語法。

  2. 清空 _config.yml 的現有資料，並新增以下組態 :
    → 老師沒清空， 就在 _config.yml 最下面有個 Deployment ，複製貼上改成自己的

  1. 執行 hexo clean && hexo deploy 。 >> 2個指令一起執行，先清除再部屬
    我的會出現錯誤，指令改成hexo clean ; hexo deploy 就 OK 了~

會讓你再登入一次。

到 GitHub 檔案頁面 https://github.com/使用者名稱/專案名稱/settings
Settings > Pages 看到已經上傳了~

  1. 瀏覽 <GitHub 用戶名>.github.io 檢查你的網站能否運作，可能要20分鐘~3小時，GitHub 才會更新。自己測試是2分鐘，但有遇過更久的。

何时使用 hexo deploy 或 hexo clean ; hexo deploy (問了 Chat GPT)

  • 使用 hexo deploy 的情況：
    當你只是做了小範圍的修改，例如修改了一些文章的內容或樣式，且你對之前生成的靜態文件沒有任何疑慮，這時直接使用 hexo deploy 就可以將修改部署到你的伺服器上。

  • 使用 hexo clean ; hexo deploy 的情況：
    當你做了較大範圍的修改，例如修改了配置文件、主題或是插件，這些改動可能會影響到生成的靜態文件結構或內容，建議先使用 hexo clean 清除之前生成的靜態文件，再使用 hexo deploy 重新生成並部署網站。這樣可以避免因為舊文件殘留而導致的潛在問題。

hexo clean: 清除 Hexo 生成的靜態文件和緩存。
hexo deploy: 生成並部署新的靜態文件。
