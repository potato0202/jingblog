Hexo + Github 免費架站筆記 (一)

發表於2024-06-16|更新於2024-06-17|Hexo
|閱讀量:

上了6角體驗課程，今年(2024)有新上架 「Hexo 部落格架站任務」，紀錄一下架站筆記跟自己架站體驗。

  • Hexo 靜態部落格，只秀東西給人看，不能購物電商，沒有動態資料庫~
  • 可以掛 google AD 廣告。
  • 使用 markdown 語法寫文章，不是像 WordPress 後臺編輯器那種。
  • 想要選單有子選單也要靠模板
  • HEXO 模板 ，建議找在 github 上星星數高的，

話說之前有在 hahow 買一堂 Hugo 架站的課，但架起來沒有繼續寫筆記就荒廢了。
有看到學員架了一個類似官網的靜態網站。
不管是 Hugo 還是 Hexo 都蠻依賴你選擇的版型(感覺 WordPress 也是)，除非自己有辦法修改。
啊如果想要什麼輪播功能也是要看版型有沒有，大部分都沒有🥲畢竟這東西初衷就是部落格用的~

進入正題~
會使用 Visual Studio Code（簡稱VS Code） 來寫 code

Hexo 部落格架站任務，先把環境架起來

先下載 node.js 安裝
在 VS Code [檢視] → [終端] 打以下指令 :

  • node -v ( -v前面有空格哦!! 會出現表示node js版本，例如 v20.14.0 )
  • npm -v
  • git -v ( 有點忘記這邊要不要先安裝 git，如果現在沒裝也可以先裝，後面會需要，到 git 官網 安裝下載 )
  • npm install -g hexo-cli ( 安裝 hexo-cli 這個 npm 套件，g 表示裝在整個電腦上，在電腦的根目錄， C 槽幫你做全站安裝 )
  • hexo -v ( 安裝成功會寫版本號，例如 hexo-cli 4.3.2 )

這時候!! 輸入完 hexo -v 出現錯誤!!

跑去看6角的體驗營 Discord 看有沒有人有同樣問題

看到解答，照這位 blog 版主 寫的操作就可以了~
文章底下那個 BUT 問題也有遇到😂

新增一個你放 blog 檔案的資料夾 與 終端機操作

  • 前往指令 : cd → 表示前往到專案資料夾
  • 終端機操作分享 : https://w3c.hexschool.com/git/8dc4619b

先在桌面或其他地方(也許D槽)新增一個資料夾，
在 VScode 輸入 cd → 空白鍵空格 → 然後將你剛剛建立的資料夾(例如:桌面的hexo0522資料夾)拖曳進來，按下enter，就OK~
上方的「終端機操作分享」有影片教學。

hexo指令

01.建立部落格 : hexo init

輸入指令hexo init 你的網站名稱(ex: coffeeshop)(hexo init後方有空白哦)，
他會在 hexo0522 資料夾裡頭出現一個 coffeeshop 資料夾。
coffeeshop 資料夾裡頭有很多 hexo 相關檔案~

我自己是在桌面建立”hexo_web”資料夾，輸入指令 hexo init lifeblog

下次要開檔案可以 「檔案」>「開啟資料夾」選擇 coffeeshop (我是lifeblog)資料夾，就不用再打 cd 指令。

02.開啟伺服器 : hexo server

輸入指令 hexo server ，http://localhost:4000/ 就是你的站，可看外觀， Ctrl + C 會關閉 server。
要關閉伺服器才能繼續打指令~

03.輸出實際網頁 : hexo generate

輸入指令 hexo generate，出現了public資料夾!!
hexo generate 編譯動作，編譯出網頁看得懂的格式。

04.新增一篇部落格 : hexo new “部落格標題”

圖中範例，輸入指令 hexo new "新文章"，就會在 source /_posts 資料夾 ( 看左側面板 ) 裡頭產生了一個新的 md 檔案叫做「新文章」，點進去編輯檔案內容文字，用 markdown 語法 ( md 其實瀏覽器看不懂，也是透過編譯才懂)

「如何寫 markdown 語法」>> 這裡看

我這邊自己在 _posts 資料夾 右鍵新增好幾個分類用資料夾，例如 : css、photoshp，
然後複製舊的 md 檔案到這些資料夾裡頭，用舊的 md 檔案改寫，測試過上傳到 github 都是可以觀看的~

05.新增頁面 : hexo new page 頁面名稱

輸入指令 hexo new page about 會新增一個 about 資料夾，裡頭有 index.md 檔。

並不是出現在_posts 資料夾裡，跟上面的hexo new "新文章"不一樣
如果是取名 about 有可能會自動出現在選單中，假設這個 md 頁面沒出現在 menu 選單中，想要這 md 頁面出現在選單，我用的hexo模板( butterfly )是要到 _config.butterfly.yml 設定檔裡頭，找到 menu 的地方手動新增。看別人套用 Next 模板也有相同操作~

06. 清除快取 : hexo clean

有遇到奇怪的問題例如沒更新到，輸入指令 hexo clean 試試看，會清除快取檔案 (db.json) 以及編譯檔案 (public)

07. 部屬網站 : hexo deploy

這個最後部署到 GitHub Page 的時候會用到~

全站設定 :

_config.yml >> 可以設定網站title、關鍵字、description這些，後續會需要修改

安裝 theme 內建樣版

讓你新增頁面，像是about時，MENU會出現項目，目前 themes 資料夾是空的(只有1個.gitkeep檔案)
樣板連結
裡頭有安裝教學~

這邊我們使用 git 安裝，

輸入指令 :
git clone --depth 1 https://github.com/hexojs/hexo-theme-landscape themes/landscape
在 themes 資料夾裡會產生 landscape 資料夾。

自己安裝遇到錯誤，要去安裝 git 就 OK 了

安裝完，輸入 git –version 可看版本。

在 themes 資料夾 → landscape 資料夾 → 有一個 _config.yml 檔案，

  • 01.更改 menu 選單，改成中文字或是新增頁面
    在#menu的地方

  • 02.寫上要使用的模板名稱，找到theme
    # Extensions
    ## Plugins: https://hexo.io/plugins/

## Themes: https://hexo.io/themes/
theme: landscape → 在 theme 後方寫上 landscape
